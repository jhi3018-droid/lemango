# 르망고 오너 매뉴얼 (Fable 기억 원장)

> **용도**: Fable(Claude.ai)의 기억 보조 문서. jjo는 평소처럼 그냥 물어보면 되고, 새 세션 시작이나 큰 작업 때 이 파일을 Fable에게 첨부하면 완전한 맥락을 장착함.
> **보관**: `lemango/docs/오너매뉴얼.md` (repo 커밋 권장 — 버전 관리 + 분실 방지)
> **갱신**: 큰 작업 완료 시 Fable이 추가 내용을 정리 → CC가 커밋
> **v1**: 2026-07-14

---

## 1. 사용법 (까먹기 쉬운 것들)

### 매장 할당 (직원 ↔ 매장)
- 인사관리에서 **"수정" 버튼 말고 직원 이메일 클릭** → 프로필 팝업에 매장 드롭다운 (grade≥4만 설정 가능)
- 매장 목록 자체는 설정 메뉴(매장명 관리)에서. 현재: 부산점(st1), 성남점(st2)
- **grade 1~2 + 매장 할당 = 그 매장만 작업** (서버 규칙 강제). grade≥3(관리자)은 전 매장 + 스위처
- 미할당 직원 = POS 작업 차단 ("배정된 매장이 없습니다")
- ⚠️ 직원을 관리자(grade≥3)로 올리면 매장 제한이 자동 해제됨

### 카페24 등록 CSV 내보내기 (상품조회)
- 흐름: 상품 체크(또는 체크 없이 = 필터 전체) → **🛒 카페24 등록 CSV** → 다운로드 → **경고 모달 확인** → 경고 품번은 상세에서 보정(색상코드/영문명/제조국/제조일자/대표이미지/상세URL) → 재내보내기 → 카페24 admin 대량등록 업로드
- 경고가 많이 떠도 정상 (레거시 데이터 공란 때문). **조용한 부실 내보내기 방지** 설계
- **제조국 비우면** → 파일엔 빈칸 → **카페24가 국내산으로 기본 처리**. 중국산은 제조국 필수 입력
- 분류번호는 자동 생성 (규칙은 §2). 매핑 불가 상품은 공란+경고 → 카페24에서 수동 지정

### 재고현황 엑셀 (매장 → 매장별 재고현황)
- **📥 엑셀 다운로드**: 세로형 `품번 | 상품명 | 사이즈 | 바코드 | 수량`. 바코드 있거나 재고 있는 사이즈만 행 생성. 바코드는 텍스트 서식(13자리 보존)
- 헤더에 **총재고** 표시 (합계 열 총합과 항상 일치)
- 엑셀 품번 수 < 화면 품번 수일 수 있음 (재고·바코드 둘 다 없는 품번은 생략) — 총재고는 일치

### 재고 엑셀 업로드 → 입출고 내역 기록
- 업로드하면 이제 **입출고 내역에 유형 "재고 업로드"로 기록됨** (증감 = 새값−이전값, 작업자, 파일명)
- 같은 파일 재업로드(변경 0) = 기록 0건. 재고 낮춘 SET은 음수 증감
- 업로드 행은 **취소 불가** — 정정은 재업로드나 재고수정으로
- ⚠️ 2026-07-14 이전에 올린 초기 재고(성남점 2,174개)는 기록 없음 (소급 금지 결정, §2) — 원본 엑셀 파일 보관이 근거

### 기준재고 설정 (🧭)
- **재고를 바꾸지 않음.** "이 시점 재고 = X" 스냅샷만 저장 (감사 앵커)
- 효과: 재고 대조(기준 + 이후 기록 = 현재고 검증)가 이때부터 작동
- 언제든 다시 찍으면 새 기준으로 갱신. 찍기 전 시스템↔실물 스팟체크 권장
- 판매가 이미 있었어도 무관 — "지금 값"을 찍는 것

### 바코드
- 사이즈 표준: **XS S M L XL 2XL F**. 업로드 파일에 XXL/2X로 써도 2XL로 자동 인식 (정확 매칭, 부분 매칭 없음)
- 바코드가 엉뚱한 사이즈에 들어가 있을 때: **같은 파일 2회 업로드** (1회차 덮어쓰기로 정정 → 2회차 신규 등록)
- 바코드 감사 콘솔 명령: `getBarcodeCollisions()` (충돌 목록), 전체 덤프는 Fable에게 문의

### 기타
- **신규기획 날짜 필터 = 등록일 기준** (라벨: 등록일(시작)/등록일(끝)). 시작일만 넣으면 그날 이후 전부
- **상품 = 품번 필수.** 품번 없이 상품 직접 등록 금지 — 신규기획에서 품번 생성 → 확정 흐름이 정상
- 콘솔의 `initAdmin skip: Missing or insufficient permissions` = **정상** (보안 가드, 무시)
- 배포: 코드 = `firebase deploy --only hosting` / 규칙 변경 시에만 `--only firestore:rules` (별도)

## 2. 결정 기록 (왜 그렇게 정했는지)

| 결정 | 이유 |
|---|---|
| **바코드 키에 색상 없음** (품번+사이즈만) | 상품 1개 = 품번 1개 = 색상 1개 구조(World A). 색상별로 품번이 다르므로 색상 키 불필요. 2026-07 검토 후 확정 |
| **소급 입고 기록 금지** | 실제 시각 모르는 기록 = 장부 조작. 대조 기능이 영구히 안 맞게 됨. 대신 기준재고 앵커 + 엑셀 원본 보관 |
| **사이즈 표기 = 2XL 통일** (XXL은 입력 별칭) | 사이즈규격이 XXL, 재고가 2XL로 어긋나 바코드 사고 발생 → 단일 소스(SIZES)로 통일, F도 부위별 |
| **카페24 상품명** = 한글명 + 색상약어 + (품번 뒤4) | 예: 프리즘 아이스 BL(6301). 약어 없으면 생략+경고 |
| **카페24 검색어** = 브랜드,한글상품명,색상명 / 수모 = 수모,뒤4자리 | 실제 등록 파일 형식 채택 (C1 결정) |
| **카페24 소재** = material 필드 (fabricDesc 아님) | fabricDesc는 전 상품 공란. 겉감:...%,\n안감:... 형식으로 변환 |
| **간략설명 헤더 = 제조국 연동** | 한국: ⭐한국 생산+📦전 공정 자체 생산 / 중국: ⭐중국 생산 (전공정 줄 제외) |
| **분류번호 규칙** | 의류 = 428\|390\|391(NEW 포함) / ACC = 428\|390(NEW 제외) + 경로. 느와 = NOIR 전용(WOMEN 미적용). 원피스는 레그컷까지(없으면 401 멈춤). BEST·콜라보 = 자동 생성 안 함(개별 수정). 매핑 불가 = 공란+경고 |
| **코드표** | 브랜드: 르망고 B0000000 / 느와 B000000D. 원산지: 한국 1798 / 중국 264. 제조사 M0000000·공급사 S0000000·트렌드 T0000000 고정 |
| **업로드 행 취소 불가** | 다중 행 배치라 부분 취소 혼란 + 원장 무결성. 위치이동과 동일 취급 |
| **문서 역할 분담** | CLAUDE.md = 기술 원장(CC용) / 오너매뉴얼 = 결정·사용법(Fable용). 중복 금지 |

## 3. 미해결 / 백로그

- **느와 굿즈 분류**: 393\|395\|439 vs 393\|439 — 실물 나오면 확인 (현재 395 포함으로 구현)
- **WH/LT 동일 바코드**: LWWAL43002017-WH와 -LT가 바코드 5개 동일 — jjo 직접 정정 예정
- **분류 미매핑 갭**: 남성 숏브리프/5"쇼츠, 고글, 서바이벌킷, 공용(N/U) — 품번 신호 없음, 경고+개별 처리 (근본 해결 = 품번 타입코드 확장, 별도 논의)
- **매장할당 UX**: "수정" 모달에 매장 드롭다운 + 직원 목록에 매장 컬럼 (기능은 있음, 발견성만)
- **업로드 오류 "품번 복사" → 엑셀 다운로드** (2번 요청 — 어느 업로드인지 미확정)
- **타입 목록에 수영모 추가** (지금은 품번 SC로 판정)
- **기획 라운드트립 export가 구 이미지 모델** (_downloadPlanFull — 상품만 신모델 전환됨)
- jjo의 "3번" 요청 — 미접수

## 4. 운영 원칙 (오너용)

- **한 번에 한 결정.** 애매하면 Fable에게 그냥 물어보기 (기억은 Fable+이 문서가 담당)
- **분기마다 "빼기 점검"**: 안 쓰는 기능 / 헷갈리는 화면 / 중복 동선 목록화 → Fable에게 전달 (기능 추가보다 다듬기)
- 실재고·원장·매출공식은 불가침 — 관련 작업은 항상 조사 → 확정 → 구현 순서
- 새 스킬/도구는 Fable이 필요할 때 먼저 제안 (jjo가 쫓아다닐 필요 없음)
