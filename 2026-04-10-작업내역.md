# 르망고 시스템 작업 내역 — 2026-04-10

## 1. 출퇴근/연차 데이터 Firestore 마이그레이션 (핵심)

### 문제
- 출퇴근/연차 데이터가 **localStorage(브라우저 내부 저장소)**에만 저장됨
- 이효정이 본인 PC에서 출퇴근 체크해도 관리자 PC에서는 보이지 않는 구조적 문제

### 해결
- 출퇴근(`attendance`), 연차(`leaves`), 연차쿼터(`leaveQuotas`) 3개 Firestore 컬렉션 신설
- 캐시 레이어 도입: Firestore에서 앱 시작 시 전체 로드 → 메모리 캐시로 읽기, 저장 시 Firestore + 캐시 동시 갱신
- 기존 localStorage 데이터 → Firestore 자동 마이그레이션 (1회성, 각 사용자가 앱 접속 시 실행)
- Firestore 보안 규칙에 3개 컬렉션 추가 배포

### 수정 파일
- `js/hr.js` — 읽기 패턴 8곳 + 쓰기 패턴 14곳 전환
- `js/main.js` — `initApp()`에 `loadHrData()` 호출 추가
- `firestore.rules` — attendance/leaves/leaveQuotas 규칙 추가

## 2. 부서 필터 안 되는 버그 수정

### 문제
- 팀원 연차현황, 팀원 출퇴근 탭에서 부서 필터가 "전체"만 표시되고 부서 목록이 비어있음

### 원인
- Firestore `users` 컬렉션에 부서가 `dept` 필드로 저장되는데, hr.js에서 `u.department`로 참조
- 필드명 불일치로 부서 목록 수집이 0건

### 해결
- `hr.js` 전체에서 `department` → `dept`로 일괄 변경 (필터, 표시, 미처리건 체크 등 전체)

## 4. 개인별 연차 상세 모달 추가

### 문제
- 팀원 연차현황 카드가 클릭 불가, 상세 정보를 볼 수 없음

### 해결
- 연차 카드 클릭 시 **연차 상세 모달** 팝업
- 상단: 총연차 / 사용 / 잔여 / 소진율 4개 요약 카드
- 하단: 연차 사용 이력 테이블 (유형, 시작일, 종료일, 일수, 상태, 사유)

### 수정 파일
- `js/hr.js` — `openLeaveDetail()` 함수 신규
- `index.html` — `leaveDetailModal` 다이얼로그 추가
- `js/main.js` — `makeDraggableResizable` 등록
- `style.css` — `.ld-*` 스타일 (요약 그리드, 테이블, 상태 배지)
